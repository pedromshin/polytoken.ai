packages/genui/src/theme/tokens.ts
- `TOKEN_ALIASES` — the closed tuple (currently 20 entries) every pack must define. z.enum in schema/token-props-schema.ts derives from it. Adding an alias here auto-extends the allowlist enum AND the pack-completeness test.

packages/genui/src/theme/packs.ts
- `makeTokens(values)` — the `satisfies Record<TOKEN_ALIASES[number], string>` gate: a pack missing a new alias fails to compile.
- `resolveVars(tokens)` — HAND-MAINTAINED, line-by-line (`vars["radius"] = tokens["radius.base"]` etc.). This is NOT compile-checked for completeness — a new alias silently omitted here is the exact "var exists but utility never registered" bug class D-48-08 targets. It MUST gain a line per new alias.
- 6 pack token maps: NAUTA_TEAL_TOKENS, LINEAR_CLEAN_TOKENS, WARM_EDITORIAL_TOKENS, BRUTALIST_TOKENS, CORPORATE_SAAS_TOKENS, PLAYFUL_ROUNDED_TOKENS — each authored with inline WCAG-AA contrast comments.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM spec output -> token allowlist | Model-authored `style_pack_id`/per-node token props cross into rendering |
| Token value strings -> CSS injection | Pack values become `hsl(var(--x))` / CSS custom properties |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48-01 | Tampering | New color aliases in packs.ts | mitigate | Values stay HSL channel-triplets; existing `HSL_TRIPLET_RE` + no-raw-hex assertions in packs.test.ts auto-apply to the new `color.*` aliases (they iterate `COLOR_ALIASES`). |
| T-48-02 | Tampering | TokenAliasSchema allowlist | mitigate | New aliases enter the closed z.enum via TOKEN_ALIASES; token-allowlist.test.ts asserts enum == TOKEN_ALIASES, so no free-form value path opens. Resolved values always come from the pack registry, never LLM output. |
| T-48-03 | Denial-of-service | `radius.pill: 9999px` / touch-target | accept | Static CSS length; no runtime computation, no attacker input. |
| T-48-SC | Tampering | npm/pip/cargo installs | accept | No package installs this phase (additive data + config + tests only); Package Legitimacy Gate not triggered. |
</threat_model>

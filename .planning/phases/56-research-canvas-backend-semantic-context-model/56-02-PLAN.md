<objective>
Wire RCNV-01's auto-collect: the instant a `web_search` tool result passes the existing FOUND-6
envelope-quarantine gate inside `_run_server_tool_round`, write one `chat_source_ledger` row per
search result — automatically, synchronously, with NO capture-confirm ceremony. Today's CLUS-04
`emit_confirm_action`/source_capture widget is the explicit anti-goal.

Purpose: satisfies RCNV-01 and the phase's Success Criterion #1 — "every tool result used during a
conversation is recorded in a per-conversation source ledger automatically, with no manual per-turn
confirmation step."

Output: a new domain port + Supabase adapter for the ledger, a fail-open write hook in the tool-round
loop, additive-default DI wiring, and a test proving the row is written (and the fail-open + regression
guarantees hold).

This plan makes ZERO knowledge-graph writes — the ledger is a separate candidate pool (999.19). It does
NOT add a settings kill-switch (A4: gating is inherited transitively from WEB_SEARCH_TOOL_ENABLED — the
hook only ever fires for an already-gated tool). It does NOT touch chat_context_edges (that is 56-03/04).
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-02-01 | Denial of Service | model turn | mitigate | The hook is fully fail-open (try/except Exception → warning → return); a malformed envelope or DB error never raises past the tool round (Pitfall 1) |
| T-56-02-02 | Tampering | envelope parse | mitigate | Parse only the ALREADY-gated envelope; skip results lacking a url; never trust unbounded future-tool envelopes (defensive json.loads try/except) |
| T-56-02-03 | Elevation of Privilege | ledger tenancy | mitigate | Rows carry conversation_id (the ownership anchor) from the live run; importer_id is denormalized audit only, never an ownership authority (Pitfall 2) |

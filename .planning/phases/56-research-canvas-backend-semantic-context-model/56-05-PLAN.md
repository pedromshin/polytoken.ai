Land the promotion-gate reuse seam: a thin (~20-line) adapter that reshapes a `chat_source_ledger` row
into the exact `source_payload` shape `SourceCaptureHandler.execute()` already accepts, calls it
verbatim, and (on success) writes the resulting node id back onto the ledger row.

Purpose: satisfies the phase's Success Criterion #3 — "the existing suggest-only promotion gate
(INFERRED → EXTRACTED) is reachable from the source ledger's records with zero new promotion code," the
seam Phase 63's canon-curation UX (RCNV-03) sits on top of.

The whole point is REUSE, not new machinery: `SourceCaptureHandler` already does the INFERRED node upsert
(dedupe via uuid5) + INFERRED edge insert, and `PromoteEdgeUseCase` already flips tier to EXTRACTED. This
plan changes NEITHER. The proof is a zero-diff assertion on those files, mirroring the existing
`test_source_capture_promote_reuse.py` (CLUS-05, 54-03).

Scope: this phase ships the use case + its reuse test + the one small ledger back-reference write. It does
NOT register the use case in DI or expose a route — Phase 63 owns the route/consumer. The reuse test
constructs the use case directly with fakes (no DI needed to prove the seam).

Output: a `set_knowledge_node_id` method on the ledger port/adapter, the new use case, and the reuse test.

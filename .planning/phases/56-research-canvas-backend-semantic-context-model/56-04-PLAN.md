<objective>
Wire RCNV-04's read/inject half: at turn time, resolve a conversation's active `chat_context_edges` to
their referenced content and inject it as a bounded, quarantined LINKED CONTEXT block into the system
prompt — a SECOND pipeline sitting ALONGSIDE (never nested inside) the existing thread/cluster
injection.

Purpose: completes RCNV-04 / Success Criterion #2 — "drawing a canvas edge from a source/table/panel
node to a chat node causes that node's content to be injected into the chat's context on the next turn."

Key architectural constraint (RESEARCH Pattern 3): this MUST NOT be nested inside
`_system_prompt_with_cluster_context`'s thread-linkage gate. A conversation with zero thread linkage must
still receive linked-edge context. It is an independent, independently fail-open call with its own budget.

## Honors Decision D-56-A (from 56-03)
A `knowledge_node`-typed edge resolves and injects REGARDLESS of the node's tier (INFERRED / AMBIGUOUS /
EXTRACTED). Read the target node DIRECTLY (a tier-agnostic get-by-id), NOT through
`KnowledgeGraphRepository.list_injectable_edges` (whose EXTRACTED-only gate governs AUTOMATIC injection,
not an explicit user-drawn edge — Pitfall 3 / Landmine 3). The DATA-never-instructions quarantine wrapper
neutralizes the trust question for this one-turn, explicitly-selected injection.

Output: a new context-edge port + Supabase adapter, a pure `linked_context.py` domain service (structural
sibling of `thread_cluster_context.py`), the second `_execute_turn` hook, additive-default DI wiring, and
tests. Deterministic tests prove the injection ASSEMBLY; the live "chat response references the injected
content" leg is a documented human-verifiable follow-up (Bedrock may be down).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stored context edge → system prompt | Resolved edge content (untrusted, incl. model-authored genui panels) crosses into the LLM prompt |
| edge read → tenant scope | The Python read has only importer_id, not a live per-request user — ownership was settled at write time (56-03) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-04-01 | Tampering / Elevation | injected content as instructions (prompt injection) | mitigate | DATA-never-instructions quarantine header FIRST + truncate_field budget, mirroring thread/cluster (Landmine 1); genui_panel content (model-authored) injected through the SAME discipline, never relaxed |
| T-56-04-02 | Information Disclosure | cross-tenant read | mitigate | Ownership is enforced at edge WRITE time (56-03 assertSourceRefOwnership); the read only trusts already-ownership-gated stored rows scoped by conversation_id (Landmine 2) |
| T-56-04-03 | Denial of Service | turn availability | mitigate | The linked-context call is fully fail-open ([] on any error / unwired collaborator); never nested in the thread gate; never raises past _execute_turn |
| T-56-04-04 | Elevation of Privilege | tier bypass concern | accept | Per D-56-A: knowledge_node injects tier-agnostically because the user's explicit edge IS the selection; quarantine wrapper neutralizes trust; distinct from list_injectable_edges's automatic-injection gate (Landmine 3) |
| T-56-04-SC | Tampering | package installs | accept | N/A — zero new dependencies (RESEARCH Package Legitimacy Audit) |
</threat_model>

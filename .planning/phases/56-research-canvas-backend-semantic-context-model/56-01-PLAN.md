<objective>
Land the palette-independent data-model foundation for the research canvas: two new, purely
additive Postgres tables authored as Drizzle schema and generated into a migration.

- `chat_source_ledger` (RCNV-01): a per-conversation, zero-knowledge-graph-write candidate pool of
  tool sources the agent used. NOT the knowledge graph — a separate table by design (999.19's own
  framing).
- `chat_context_edges` (RCNV-04): the durable semantic linkage store for "this canvas node's content
  is context for this chat," replacing canvas `sharedState` which D-54 explicitly ruled out.

Purpose: every downstream plan in this phase (the Python write hook, the tRPC edge seam, the Python
read/inject pipeline, the promotion adapter) reads or writes these two tables. They must exist and be
schema-typed first.

Output: two new Drizzle schema files, a barrel update, and one generated migration (SQL + snapshot +
journal entry). No behavior change — DDL only.

Scope note: this plan does NOT apply the migration to any environment (Docker/Supabase may be down —
same posture as Phase 54's 0036). "Authored + generated + schema-typed" is the deliverable; "applied
local→staging→prod" is a documented later step, exactly the 54-01 distinction.
</objective>

> Visual and interaction contract for the total re-skin. Generated by gsd-ui-researcher, verified
> by gsd-ui-checker. **This phase is a refinement pass, not a redesign (D-49-01/D-49-02 in
> 51-CONTEXT.md).** It consumes the token system Phase 48 closed — it does not add, rename, or

## Typography

Inherited, unchanged (2-weight ceiling locked by `product-register-and-bans.md`'s product
register). This phase does not introduce a new size or weight — it only fixes cases where a
component drifted onto a non-standard combination.

| Role | Size | Weight | Line Height (Tailwind default for the utility) |
|------|------|--------|-------------|
| Label / meta | 12px (`text-xs`) | 400 (`font-normal`) | 16px (1.33) |
| Body | 14px (`text-sm`) — dominant size across chat/inbox/knowledge chrome | 400 (`font-normal`) | 20px (1.43) |

only the palette-class conversion named in D-49-03/D-49-06.

## Color

**Naming disambiguation (read first):** shadcn's own `--accent`/`accent-foreground` CSS variable
in this app (`hsl(164 10% 94.3%)`) is a **near-neutral hover/highlight surface**, NOT the "10%
brand accent" in the classic 60/30/10 sense — it's the neutral/ghost hover destination in
`hover-active-convention.md`. The 60/30/10 "accent" role below refers to `--primary` (the
polytoken teal-green brand color), which is a **separate** token. Do not confuse the two when
implementing.

| Role | Token / Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` `hsl(0 0% 100%)` (light) / `hsl(0 0% 3.9%)` (dark) → `bg-background`, `text-foreground` | Page background, canvas background, default text |
| Secondary (30%) | `--card`/`--secondary`/`--muted` (`hsl(0 0% 100%)`/`hsl(164 8% 93.5%)`/`hsl(164 6% 95.3%)`) → `bg-card`, `bg-secondary`, `bg-muted` | Cards, sidebar (`bg-sidebar` aliases to `--background`), nav, panel rails, muted rows |
| Accent (10%) | `--primary` `hsl(164 39% 22%)` (polytoken teal-green) → `bg-primary`, `text-primary`, `border-primary` | **Reserved for:** primary CTA buttons (Send, Sign in with Google, Confirm/primary submit), active/selected state (`conversation-row.tsx` `isActive`, active canvas-node selection ring, active nav item), the generation-in-progress ring (`.generating-ring`, ADOPT-03), the `BrandMark` glyph (`text-primary`). **Never** used as a resting/decorative fill, never as the hover destination for neutral elements (that's `--accent`, see disambiguation above), never on more than one focal element per view. |
| Destructive | `--destructive` `hsl(0 84.2% 60.2%)` → `bg-destructive`, `text-destructive`, `border-destructive` | Irreversible/delete actions ONLY (delete conversation, unmerge entity, deny/reject). Stop/cancel controls use `muted`, never `destructive` (locked convention, held since 48-03 — do not relabel). |
| Success | `--success` `hsl(142 71% 29%)` (D-48-02) | Confirm/positive actions (`ConfirmDenyControls` confirm button, promotion-confirmed states). Never used to relabel a stop/deny action. |
| Focus ring | `--ring` `hsl(0 0% 3.9%)` (light) — a **separate neutral token**, not `--primary` | `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-1` on every interactive element. Orthogonal to hover/active (per `hover-active-convention.md` §1) — never dropped or reinterpreted by this phase. |

## Interactive-State Contract (D-48-06, applies to every touched element)

Governed entirely by `docs/design/hover-active-convention.md` — this UI-SPEC does not restate
values, it points to it as the binding contract:

- **Neutral/ghost** (chips, muted rows, outline buttons, unselected filter segments): resting
  `bg-muted`/`bg-background` + `text-muted-foreground` → hover moves to `hover:bg-accent
  hover:text-accent-foreground`.
- **Filled semantic** (primary/success/destructive fills): resting `bg-{alias}` +
  `text-{alias}-foreground` → hover `hover:bg-{alias}/90` → active `active:bg-{alias}/80`.
- **Focus-visible** is orthogonal, always present: `focus-visible:ring-2
  focus-visible:ring-ring focus-visible:ring-offset-1` (or `-2`).
- **One documented exception**: a persistent selected/pinned state (e.g.
  `tier-filter-control.tsx`'s active segment) suppresses the hover intensify step — hover
  reasserts the same active values rather than going further. Do not invent a second exception
  without updating the convention doc itself (out of scope this phase).

Every newly re-skinned interactive element in this phase must be classified into one of the two
families above before any hover/active class is written.

## Copywriting Contract

Voice register locked by `docs/design/brand-guide.md` §2: warm, human, companion; first-person
framing; never infrastructure vocabulary ("node," "pipeline," "daemon," "compute," "mesh") in
user-facing copy; never a meta-critical aside (ban #13). This phase does not invent new copy
except where a straggler still carries the old systems-register tone — apply the Do/Don't table
below verbatim; it is already a locked decision, not a discretion area.

| Element | Copy |
|---------|------|
| Primary CTA — Login | "Pick up right where you left off — sign in with Google." (button label: "Sign in with Google") |
| Primary CTA — Chat composer | Icon-only send affordance (no new label needed; re-skin the button chrome only, do not add a text label) |
| Login card title | "Welcome back to your workspace" |
| Chat empty-state heading | "Ask me anything" |
| Canvas empty-state heading | "Panels will appear here" |
| Email reprocess success toast | "On it — reprocessing this email" |
| Error state (pattern, exemplar `inline-error-card.tsx`) | Problem statement in plain language + a concrete next step as a real control, never just a dismiss: "Something went wrong generating this response." + `Retry` button. Apply the same problem+action shape to any other error surface touched this phase (never soften urgency, never a meta-critical aside). |
| Destructive confirmation — delete conversation (exemplar, unchanged copy) | Title: "Delete this conversation?" Body: "This permanently deletes all messages in "{title}". This can't be undone." Actions: Cancel = "Keep conversation" / Confirm = "Delete" |
| Destructive confirmation — unmerge entity (exemplar, unchanged copy) | Title: "Unmerge this entity?" Actions: Cancel / Confirm = "Unmerge" |

This phase's job on copy is a **register check**, not new copy authorship — confirm every
re-skinned surface's visible strings match this table's tone (warm, first-person, no
infrastructure vocabulary) and fix any straggler found; do not invent new empty/error/CTA copy
beyond what's listed here unless a surface genuinely has none today, in which case follow the
Do/Don't table's pattern (see `brand-guide.md` §2) and note the addition in the plan.

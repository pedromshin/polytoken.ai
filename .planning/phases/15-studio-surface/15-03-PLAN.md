actions, empty mutate seam), right = `<pre>{JSON.stringify(spec,null,2)}</pre>` — the exact object rendered.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → tRPC genui.generate | The intent + optional raw content cross to the server-side procedure (which proxies to FastAPI with the server-only API key) |
| tRPC → client | Only the re-validated spec + enum outcome + boolean cacheHit return; the spec is rendered via the trusted interpreter |
| generated spec → SpecRenderer | The validated SpecRoot is data rendered by createElement; actions resolve via the allowlisted ActionRegistry |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-09 | Tampering | renderer reuse (STDO-02) | mitigate | The sandbox preview mounts the EXACT shared `SpecRendererIsland` (production SpecRenderer + default COMPONENT_REGISTRY) on the validated spec from genui.generate. grep asserts no second renderer/registry in apps/web. The web-boundary `SpecRootSchema.safeParse` (shipped, 15-01) is authoritative before the spec reaches the island. |
| T-15-10 | Tampering | render sink | mitigate | No `eval`/`Function`/`dangerouslySetInnerHTML` in any studio file (grep gate across all studio files). The spec is plain data; JSON pane is `<pre>{JSON.stringify(...)}</pre>` (no HTML injection). D-15/GR-01/SPEC-02. |
| T-15-11 | Elevation of privilege | sandbox actions | mitigate | Actions wired via `buildActionRegistry` (query/setState/navigate only); the `mutate` seam stays empty (SEAM-02 — grep asserts no mutate handler). navigate keeps its D-15 runtime relative-href re-check. No new action surface. |
| T-15-12 | Spoofing | EMAIL_LISTENER_API_KEY | mitigate | genui.generate is a server-side tRPC procedure; the key is read server-side via getListenerConfig() and never NEXT_PUBLIC_. The sandbox client only calls the procedure — grep asserts no secret in client modules. |
| T-15-13 | Information disclosure | error/reason copy | accept | The `reason` shown in the fallback banner is the existing T-13-19 friendly, detail-free string from genui.generate — no internal detail leaks. The four-state signals are enum/boolean only. |

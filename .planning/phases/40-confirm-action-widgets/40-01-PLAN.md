Add the `emit_confirm_action` terminal widget tool: the model supplies only a `suggestionRef {kind, id}` (+ optional short rationale); the server re-reads the live `knowledge_node_edges` row at emission time and either builds a frozen confirm/reject widget declaration or fails into visible text. Extends the `chat_widget_interactions.widget_kind` CHECK constraint via a new migration.

Purpose: retrofit v1.5's edge-promotion confirm as a Phase-24-style chat widget without the LLM ever touching mutation parameters (CONF-01).
Output: a working `emit_confirm_action` tool wired into `RunChatTurn`, a widget_kind migration, and tests proving the live-read validation gate (available/unavailable/malformed).

Add a NEW trailing param `match_subject_entity_instance_id uuid DEFAULT NULL` to BOTH
(backward-compatible). Wrap the existing SELECT so the outer WHERE adds:
    AND (match_subject_entity_instance_id IS NULL OR NOT EXISTS (
      SELECT 1 FROM component_entity_candidate_links l
      WHERE l.was_dismissed = true AND (
        (l.component_id = match_subject_entity_instance_id AND l.entity_instance_id = <candidate id>)
        OR (l.component_id = <candidate id> AND l.entity_instance_id = match_subject_entity_instance_id))))
For the trgm RPC the candidate id is `sub.id`; for the embedding RPC add the NOT EXISTS in
the WHERE against entity_instances.id. Keep every existing filter (importer_id, entity_type_id,
source='email_extracted', is_active) and the GIN indexes UNCHANGED.

component_entity_candidate_links, dedup links store an ENTITY_INSTANCE id in the
`component_id` column, and dismiss_candidate_link writes BOTH orderings
((component_id=subject, entity_instance_id=target) AND (component_id=target,
entity_instance_id=subject)). The NOT EXISTS MUST check both orderings or a dismiss
recorded in one direction resurfaces from the other.

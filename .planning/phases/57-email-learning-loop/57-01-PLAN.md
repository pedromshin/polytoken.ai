<objective>
Close LEARN-01 gap 1: `SetComponentEntityTypeUseCase` currently overwrites a
component's `entity_type_id` with zero audit trail (`set_component_relationship.py:100`).
This plan captures every genuine reclassification as a structured, addressable
`entity_type_corrections` row using the repo's own load-before-mutate idiom (D-16),
and ships the trgm retrieval primitive (table + RPC + repo `find_similar`) that
Plan 57-02 consumes as few-shot signal.

Purpose: Satisfies ROADMAP success criterion 1 — corrections become a durable,
addressable record instead of a one-off that leaves no trace. Provides the capture
foundation the rest of Phase 57 reuses; nothing here auto-decides.

Output: New `entity_type_corrections` table + importer-scoped trgm RPC, a domain
entity + port + Supabase repository, and the capture hook wired into the existing
entity-type setter — all suggest-only, all best-effort, all tenant-scoped.

Decisions locked for this phase (resolving RESEARCH open questions — see 57-RESEARCH.md):
- **Q1 → capture BOTH pre-confirm overrides and post-confirm reclassifications.**
  Capture is keyed on `previous_entity_type_id is not None AND previous != new`, NOT
  on `extraction_status`. Premature filtering at capture time is unrecoverable; a
  retrieval-time filter can be added later if the pre-confirm signal proves noisy.
- **Q2 → the correction retrieval RPC is trgm-only** (no vector arm, no new embed
  call). This preserves the classifier's one-Bedrock-call-per-document reliability
  constraint. Consumed in Plan 57-02.
- Assumptions A1 (dedicated table, not a column on email_components / a reuse of
  extraction_records.corrected_fields) and A3 (only genuine reclassification counts)
  are adopted as written.
</objective>

domain ports/entities (lint-imports enforced; do NOT import app.infrastructure here):

  class SetComponentEntityTypeUseCase:
      def __init__(self, *, components: ComponentRepository) -> None: ...
      async def execute(self, *, component_id: str, entity_type_id: str | None,
                        importer_id: str | None = None) -> Component:
          component = await self._components.find_by_id(component_id)   # load first
          # D-18 tenant guard: importer_id derived from component row
          updated = await self._components.update_entity_type(component_id, entity_type_id)
          return updated

The Component domain entity (component.py) carries: id, email_id, importer_id,
content_text, entity_type_id (nullable), extraction_status, role — all frozen.

The load-before-mutate capture idiom to mirror (confirm_region.py: capture the prior
value on the SAME logical action BEFORE the mutation; best-effort — a capture failure
must never block the human action, exactly like confirm_region's synthesis hook).

The retrieval-RPC shape to mirror for match_entity_type_corrections_by_trgm — the
existing confirmed-region trgm RPC (migration 0009 match_components_by_trgm) JOINs
email_components to read content_text and filters match_importer_id on EVERY row.
CRITICAL difference (Pitfall 4): the correction RPC is importer-scoped ONLY — it has
NO match_entity_type_id parameter, because it runs BEFORE the type is known. It must
return corrections across ALL entity types, each tagged with its corrected type slug.

RLS pattern for a hard-FK importer-descendant table (migration 0034, verified):
  ALTER TABLE "<t>" ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "<t>_owner_authenticated" ON "<t>" TO authenticated
    USING (importer_id IN (SELECT id FROM importers WHERE user_id = auth.uid()))
    WITH CHECK (importer_id IN (SELECT id FROM importers WHERE user_id = auth.uid()));

Container wiring today: `provider.provide(SetComponentEntityTypeUseCase)`
(container.py:1091) auto-injects only ComponentRepository. Adding an optional
collaborator requires either a factory OR binding the new repo to its port and giving
the use case an optional param dishka can fill. Repository adapters are bound to ports

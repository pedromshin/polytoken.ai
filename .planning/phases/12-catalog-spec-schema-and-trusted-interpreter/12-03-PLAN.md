<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| spec tree → React render | The interpreter walks an untrusted-shaped spec tree and constructs React elements. This is THE renderer path; the no-eval guarantee lives here. |
| spec object keys → property access | `resolveDataRef` and the props filter access object keys derived from spec strings — must not reach the prototype chain or execute. |
| node.type → component dispatch | Only registry-keyed types may reach a real component; everything else hits the safe placeholder. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-09 | Elevation of Privilege | renderNode (code execution) | mitigate | No `eval`/`new Function`/`Function(`/`dangerouslySetInnerHTML` on the renderer path — rendering is `React.createElement`/JSX only. Verified by grep gate over renderer/registry/catalog returning zero matches (SPEC-02, GR-01, success criterion 2). The grep output is recorded in the SUMMARY. |
| T-12-10 | Tampering | prototype pollution via spec keys (`__proto__`/`constructor`) | mitigate | The props filter uses `Object.fromEntries(Object.entries(node).filter(...))` then `propsSchema.safeParse` with `.strict()` (Plan 01/02) — unknown/`__proto__` keys are stripped/rejected before reaching the component; `resolveDataRef` walks with `obj?.[key]` only on already-validated `ctx.data`/`ctx.state` scopes, never on raw spec, and returns `undefined` for missing keys (no prototype traversal that executes). |
| T-12-11 | Denial of Service | malformed node crashing the surface | mitigate | Every registry dispatch is wrapped in `NodeErrorBoundary` (getDerivedStateFromError) + a pre-render `safeParse` fallback; one bad node renders `NodeErrorFallback`, siblings continue (SPEC-03). Schema bounds (Plan 01 D-24) cap recursion depth/count upstream. |
| T-12-12 | Tampering | unknown/hallucinated component type | mitigate | `ctx.registry[node.type]` miss → `UnknownComponentPlaceholder`, never a thrown error or undefined component (D-06/Pitfall 2). |
| T-12-13 | Elevation of Privilege | button action → arbitrary handler | mitigate | `action` is a string ID resolved against `ActionRegistryContext` (EMPTY this phase) via `useActionRegistry`; the spec can only REFERENCE an action name, never supply a handler. Unmatched action → no-op. The mutation-allowlist path exists; real handlers wire in Phase 13 (SEAM-02). |
| T-12-SC | Tampering | npm installs (@testing-library/react if added) | mitigate | If a test-only library is added it is a well-known package (verify on npmjs.com before install). No new RUNTIME third-party dependency. Renderer ships on react + zod (existing). |

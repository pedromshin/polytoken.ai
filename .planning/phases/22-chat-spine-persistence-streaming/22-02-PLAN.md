<objective>
Build the multi-provider model system (D-04..D-07, D-09): a single `ChatProvider` port with typed
stream deltas, a curated best-in-class model registry (FOUND-2 shape) carrying transport, execution
locus, pricing and capability flags, and two real adapters — Bedrock Anthropic (existing IAM, no
key) and OpenRouter (single `OPENROUTER_API_KEY`) — both streaming deltas AND capturing real token
usage (D-22). Expose the registry via `GET /v1/chat/models` so the picker is honest.

Purpose: the transport foundation the chat agent (22-06) and SSE endpoint (22-07) sit on. Designing
the port against TWO real transports at once keeps the abstraction genuinely provider-agnostic and
leaves a clean seam for a later self-hosted OpenAI-compatible endpoint (D-07) — no rework.

Output: port, registry, two adapters, a models endpoint, settings + DI wiring, unit tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server → OpenRouter API | OPENROUTER_API_KEY leaves the process only in the Authorization header, server-side |
| LLM output → application | streamed deltas (text + tool json) are untrusted content (FOUND-6) |
| Browser → GET /v1/chat/models | untrusted client requests the registry over the X-API-Key gate |
| Package supply chain | new transport code (no new pip package — httpx already vetted) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-06 | Information disclosure | OPENROUTER_API_KEY leaking to client/logs | mitigate | key read only via settings property server-side; never NEXT_PUBLIC_; Authorization header only; body/errors logged without the key |
| T-22-07 | Spoofing | unauthenticated /v1/chat/models scrape | mitigate | require_api_key dependency on the /v1/chat router (fail-closed X-API-Key) |
| T-22-08 | Tampering | malicious tool-call json in the stream | mitigate | deltas are inert here (surfaced as ToolCallDelta text); the schema gate is enforced downstream at the web boundary (FOUND-6, 22-09) |
| T-22-09 | Denial of service | a stalled provider stream hangs the worker | mitigate | asyncio inactivity timeout rescheduled per event; StreamEnd on stall |
| T-22-SC | Tampering | supply-chain: transport dependency | accept | OpenRouter uses httpx (already a project dependency, no new install); Bedrock uses the existing anthropic client — no ASSUMED/SUS package introduced in this plan |
</threat_model>

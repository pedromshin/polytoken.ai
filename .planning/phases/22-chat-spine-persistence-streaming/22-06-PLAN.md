Build the chat agent/run orchestration (SEAM-04) and the Python-side persistence writes: a
`RunChatTurn` use case that assembles token-budget-trimmed history (D-26), routes to the right
transport via the registry, gates the turn through the cost circuit breaker (fail-closed pre-turn +
mid-stream abort), streams the provider deltas as typed run events (SEAM-03, D-27), and persists the
user + assistant messages as canonical interleaved typed parts (FOUND-1) plus the run and its
append-only events. Handles cancellation (stop, D-15), mid-stream cost cap (D-21), failure (D-19),
and regenerate-as-sibling (D-16); minimal neutral persona (D-01).

Purpose: the heart of the phase. Built and tested as an async generator with no HTTP dependency so
the SSE transport (22-07) is a thin wrapper.

Output: chat repositories, the provider router, the RunChatTurn agent, DI wiring, unit tests.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User text → agent prompt | user_text is untrusted; it forms the turn but must not be able to exfiltrate tools/data (D-03 — no data tools offered) |
| Provider output → persistence | streamed deltas are untrusted content stored as parts (FOUND-6) |
| Turn → cost enforcement | the agent must honor the breaker's block/abort verdicts, not bypass them |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-20 | Elevation of privilege | prompt-injection reaching data/tools | mitigate | no Nauta data tools offered in Phase 22 (D-03); tools list empty (emit_ui_spec added capability-gated in 22-07); minimal neutral persona (D-01) |
| T-22-21 | Denial of service / cost | agent ignoring the cost gate | mitigate | run() must call check_pre_turn before any provider call and should_abort mid-stream; tested fail-closed |
| T-22-22 | Repudiation | lost/altered turn history | mitigate | append-only run events + immutable persisted parts; every terminal state writes one terminal event (FOUND-5 provenance via run_id) |
| T-22-23 | Tampering | context poisoned by inactive/stale siblings | mitigate | history assembly filters is_active=True only (D-16) and trims to context budget (D-26) |
</threat_model>

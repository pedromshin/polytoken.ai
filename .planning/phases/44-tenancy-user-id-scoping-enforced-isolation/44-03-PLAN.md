<objective>
Close the FastAPI half of the cross-tenant boundary: make user-scoped endpoints require the BFF-forwarded `X-User-Id` (Phase 43 seam) and validate importer ownership in the repository/service layer — never trusting a client-supplied importer_id for scoping. Fix targets named in research: `emails.py:113` (query-param importer_id), `emails.py` detail/download/reprocess (no ownership gate), and `knowledge_edges.py` / `promote_edge.py` (body importer_id trusted as the only tenant check).

Purpose: FastAPI reaches Postgres via the service_role key (RLS-bypassing), so the repository/service layer IS the wall for the Python surface.
Output: an enforcing `require_user_id` dependency, owned-importer scoping in the email + promote paths, and pytest regressions locking each.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Next BFF → FastAPI | X-User-Id arrives as trusted server-to-server transport alongside X-API-Key; presence is guarded, ownership is verified in repos/services |
| client body → FastAPI | importer_id in request bodies/query params is untrusted input, never an auth claim |
| FastAPI → Postgres | service_role key bypasses RLS — the repository/service layer is the enforcement wall |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-44-03-01 | Information Disclosure | list_emails importer_id query | mitigate | Scope to owned importers resolved from X-User-Id; a non-owned importer_id yields 403/empty, never other users' rows |
| T-44-03-02 | Elevation of Privilege | get_email/download/reprocess | mitigate | Ownership assertion on email.importer_id ∈ owned set; 404 fail-closed (no existence oracle) |
| T-44-03-03 | Tampering | promote_edge body importer_id | mitigate | User-ownership guard before write; body importer_id validated against the edge's owned importer |
| T-44-03-04 | Spoofing | missing X-User-Id | mitigate | require_user_id → 401 on absent/empty header |
| T-44-03-SC | Tampering | no package installs | accept | No new pip packages added |

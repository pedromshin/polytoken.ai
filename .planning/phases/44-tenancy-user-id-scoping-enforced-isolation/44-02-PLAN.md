## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller (tRPC/web route) → ownership helper | callers pass a userId (from ctx.user, server-verified) and a client-supplied resource id; the helper must never trust the resource id for scoping without verifying ownership |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-44-02-01 | Elevation of Privilege | assert* functions | mitigate | Fail-closed: missing id and other-user id both throw OwnershipError (no "not found" vs "forbidden" oracle beyond the caller's own mapping) |
| T-44-02-02 | Tampering | Drizzle queries | mitigate | Parameterized eq/and/inArray builders only — no string interpolation |
| T-44-02-03 | Spoofing | userId source | accept | Helper trusts the userId parameter; callers are contractually required to source it from server-verified ctx.user (enforced in Plan 05/06/07), never from client input |
| T-44-02-SC | Tampering | no package installs | accept | No new npm packages added |
</threat_model>

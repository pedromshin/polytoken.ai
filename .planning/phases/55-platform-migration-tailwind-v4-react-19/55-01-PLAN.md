<objective>
Stage 1 of the five-stage Tailwind v4 + React 19 migration (55-RESEARCH.md "Recommended Migration Sequencing"): swap the build engine ONLY. Install the v4 engine, replace the three `@tailwind` directives with `@import "tailwindcss"`, swap the PostCSS plugin, and keep the existing v3 JS theme alive through a transient `@config` bridge so nothing visual changes yet. Also stand up the executable regression guard that later stages depend on.

Purpose: Isolate the engine swap from the token/value port (Stage 2). If the build breaks here, it is the plugin/import plumbing — a loud, immediate, low-risk failure — not the oklch conversion. This is the lowest-risk stage per the RESEARCH stage-by-stage gate map.

Output: apps/web + packages/ui on `tailwindcss@^4.3.2`; `@import "tailwindcss"` + `@config` bridge in globals.css; the `@tailwindcss/postcss` plugin; a committed `token-render.spec.ts` computed-style guard that goes green on this (still-HSL) stack and protects Stages 2-5.
</objective>

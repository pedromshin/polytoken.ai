<objective>
Assemble and inject the cluster context that makes a thread-linked chat aware of
its thread and its siblings. At turn time, when `conversation.thread_id` is set,
build a BOUNDED, quarantine-enveloped thread-context block (subject, participants,
recent bodies within a token budget) plus a metadata-first cluster-context block
(sibling conversation titles/summaries, captured source titles+URLs, genui panel
titles) and inject it into the turn. Email and web content are untrusted — same
envelope discipline as tool results.

Purpose: CLUS-02 (agent draws on the thread's content) + CLUS-06 (cluster context
accumulates across chats). Depth-first: one bounded, safe injection path.
Output: a pure `thread_cluster_context` domain service, a thread_id-aware
repository read (feature-detected), and the RunChatTurn wiring.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| email/web content → model context | thread bodies + captured sources are untrusted; they must enter as labeled DATA, never instructions |
| thread cluster → tenant | sibling conversations + captured sources must be scoped to the caller's importer/user |
| injected block → turn budget | the block must be bounded so it can't blow the turn budget |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-54-05-01 | Tampering (prompt injection via email) | thread-context block | mitigate | labeled untrusted-DATA wrapper + hardening-line framing; assembler never interpolates content as instructions; injection-inertness test |
| T-54-05-02 | Information disclosure | cross-tenant cluster reads | mitigate | every read scoped to importer_id/user_id; sibling/source reads never trust thread_id alone |
| T-54-05-03 | Denial of service | oversized thread/cluster | mitigate | combined token budget + per-field truncation + metadata-first reservation |
| T-54-05-04 | Availability | unapplied 0036 (missing column) | mitigate | get_thread_id feature-detects; fail-open skip, warning logged, turn unchanged |
| T-54-05-05 | Tampering | mutate history | accept | injection is read-only assembly; provider_messages built immutably (spread), never mutated in place |
| T-54-05-SC | Tampering | npm/pip installs | accept | no new packages this plan |
</threat_model>

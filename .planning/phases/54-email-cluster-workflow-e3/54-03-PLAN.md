<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| model → capture proposal | model may only reference a server-assigned web_search result id; it never authors the persisted node content |
| confirm → write | nothing is written until the user's confirm; reject writes nothing |
| capture → tenant | a capture must land in the caller's own importer/cluster only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-54-03-01 | Tampering | source payload origin | mitigate | source re-read server-side from the persisted web_search result part by id — model text never trusted |
| T-54-03-02 | Elevation | capture without consent | mitigate | suggest-only via confirm-action; write happens only on user confirm; reject is a pure no-op audit |
| T-54-03-03 | Information disclosure | cross-tenant capture/promote | mitigate | importer_id/user_id scoping on upsert/insert/promote; unavailable cases collapse to one generic text |
| T-54-03-04 | Tampering | mutate-in-place | mitigate | supersede-never-mutate: find_active_node reuse + INFERRED edge, promotion via CAS |
| T-54-03-05 | Repudiation | dispatch failure after CAS | mitigate | handler never raises past execute(); returns typed status; provenance retained on the edge |
| T-54-03-SC | Tampering | npm/pip installs | accept | no new packages this plan |
</threat_model>

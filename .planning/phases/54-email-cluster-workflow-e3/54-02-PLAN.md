<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| model → web_search args | model-authored query is untrusted; top-N/timeout/size are server-fixed, never model-read |
| web_search → the internet | outbound fetch can be steered at internal hosts (SSRF) |
| fetched page → model context | page content is untrusted (prompt injection) and must stay quarantined |
| executor → run loop | executor output must satisfy the envelope gate before entering context |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-54-02-01 | Information disclosure / Elevation (SSRF) | fetch pipeline | mitigate | is_public_https_url + post-DNS is_public_ip reject private/loopback/link-local/CGNAT/non-https before any fetch (url_safety, Task 1) |
| T-54-02-02 | Tampering (prompt injection) | fetched page content | mitigate | quarantine envelope + adversarial fixture suite proves payloads stay inert; tool-result-hardening system line already appended on tool-round turns |
| T-54-02-03 | Denial of service | oversized/slow pages | mitigate | bounded read size, per-fetch timeout, top-N cap, cap_tool_output whole-envelope cap |
| T-54-02-04 | Elevation (dark exposure) | tool registration | mitigate | WEB_SEARCH_TOOL_ENABLED code-gate; enabled only after adversarial suite passes (structural omission from mappings) |
| T-54-02-05 | Repudiation | executor failures | mitigate | never raise past execute(); is_error results logged via structlog; friendly visible text |
| T-54-02-SC | Tampering | npm/pip installs | accept | no new package — httpx already a dependency; keyless DuckDuckGo needs no SDK/API key |
</threat_model>

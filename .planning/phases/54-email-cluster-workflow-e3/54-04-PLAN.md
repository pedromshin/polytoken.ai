<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| node.data → render | node.data carries only a threadId ref (.strict), never fetched content — content rehydrates via the ownership-scoped tRPC query |
| picker → attach | attach flows through the ownership-scoped mutation (54-01), not a client-trusted write |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-54-04-01 | Tampering | node.data payload | mitigate | EmailThreadNodeDataSchema `.strict()`, threadId uuid only — no content smuggled into the layout |
| T-54-04-02 | Information disclosure | threadCard fetch | mitigate | server-side ownership scope (54-01); the node cannot fetch a foreign thread |
| T-54-04-03 | Spoofing | Attach chat | mitigate | attachConversationToThread asserts conversation ownership server-side (54-01) |
| T-54-04-04 | UX/Accessibility | touch targets | mitigate | pointer-coarse: recipes on remove/footer per UI-SPEC Component 6 |
| T-54-04-SC | Tampering | shadcn/registry install | accept | no new vendored component — all primitives (Popover/Command/Tooltip/Button/Skeleton) already in @polytoken/ui |
</threat_model>

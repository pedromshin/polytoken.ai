<objective>
Prove LIVE-01's end-to-end green path is real by driving it with Playwright against the live local
stack and backing every step with a DB assertion. Because interactive Google sign-in is user-gated
(that is LIVE-03, on the DEPLOYED app), the local spec seeds an authenticated Supabase session
programmatically via the GoTrue admin API and injects the `sb-*` auth cookies — never clicking
through Google. The result is a repeatable, autonomous, DB-verified proof that login → inbox →
thread → email detail → chat (with tool rounds) → genui panel → /knowledge all work together.

Purpose: "verified against the DB not the terminal" is the explicit LIVE-01 bar; a pixel-only pass
would hide a broken wire (e.g. the fresh-DB grant gotcha silently 403-ing tRPC).
Output: seed-session helper, green-path spec, captured DB-verification artifact.
</objective>

  NEXT_PUBLIC Supabase vars; local supabase containers are up). Next dev serves on http://localhost:3000.
- vitest only collects `src/**/*.test.{ts,tsx}` (apps/web/vitest.config.ts) — the `e2e/*.spec.ts` files are NOT
  picked up by vitest, so there is no runner collision to fix. Keep the Playwright specs under `apps/web/e2e/`.

Spec 1 — apps/web/e2e/code-island-isolation.spec.ts: imports `{ ISLAND_SANDBOX, buildIslandSrcdoc }` from
`@polytoken/genui/sandbox` (a workspace subpath export → packages/genui/src/sandbox/index.ts, raw TS). Uses
`page.setContent(...)` only — NO dev server needed. 5 tests (opaque origin, parent-DOM SecurityError, cookie/
localStorage block, CSP connect-src egress block, top-nav block). MUST pass on chromium AND firefox.
INTEGRATION RISK to pre-empt: Playwright's esbuild loader must resolve the `@polytoken/genui/sandbox` exports
subpath to raw TS through the workspace symlink — verify this import resolves on both engines (this is the most
likely first-run failure).

Spec 2 — apps/web/e2e/auth-redirect.spec.ts: does `page.goto("/chat")` and asserts a redirect to
`/login?redirectTo=/chat`. REQUIRES a running dev server (uses baseURL + relative goto). No sign-in performed —
the middleware redirect fires without any OAuth session, which is exactly why it is in scope.

Config requirements: `testDir: "./e2e"`, `use: { baseURL: "http://localhost:3000" }`, `projects` for chromium
and firefox (devices["Desktop Chrome"]/["Desktop Firefox"]), and `webServer: { command: "npm run dev",
url: "http://localhost:3000", reuseExistingServer: true, timeout: ~120s }` (command runs with cwd = the config
dir = apps/web, so `npm run dev`'s dotenv path `../../.env.local` still resolves to root).

## Seam Connection Verdicts

### Seam 1 -- Phase 12 grammar -> Phase 13 generator (one source of truth)

WIRED

packages/genui/artifacts/spec.schema.json is emitted from Phase 12 Zod source via generateSchema(). Phase 13 Python generator loads it at import time via load_spec_schema() in apps/email-listener/app/infrastructure/llm/genui_artifacts.py with @lru_cache. GenuiGeneratorAdapter calls load_spec_schema() at lines 33, 151, and 268 of genui_generator_adapter.py and uses the schema as input_schema for Bedrock forced tool-use. No duplicate grammar definitions found.

### Seam 2 -- REGISTRY_VERSION -> Phase 14 cache key

WIRED

REGISTRY_VERSION (SHA-256 content hash, shape { catalogId: global, version: 64-hex }) is exported from packages/genui/src/catalog/registry-version.ts. The tRPC genui.generate procedure imports it (packages/api-client/src/router/genui/generate.ts line 35) and sends registry_version: REGISTRY_VERSION.version in the FastAPI request body (line 126). FastAPI passes it to compute_cache_key(..., registry_version=registry_version, ...) in apps/email-listener/app/application/use_cases/cache_key.py lines 111-144, where it is hashed as a required field.

### Seam 3 -- REGISTRY_VERSION -> Phase 13 prompt artifact

WIRED

packages/genui/artifacts/genui-prompt.json is committed with registryVersion: { catalogId: global, version: 64-hex } baked in. This is loaded by load_prompt_payload() at import time. Since the version is embedded at artifact-generation time via the same SHA-256 computation, the prompt artifact and the live REGISTRY_VERSION export are always in sync within a given build.

### Seam 4 -- tRPC genui.generate -> Phase 15 studio sandbox

WIRED

apps/web/src/app/studio/_components/generation-sandbox-island.tsx calls api.genui.generate.useQuery at line 96. The tRPC router is connected: packages/api-client/src/root.ts wires genui: genuiRouter at lines 4 and 13. The tRPC route handler proxies to FastAPI Python POST /v1/genui/generate. Full chain: studio -> tRPC -> FastAPI -> use case -> Bedrock/cache -> response -> studio renders. SpecRootSchema.safeParse re-validates the FastAPI response at line 218 of generate.ts (D-08 boundary re-validation), so malformed model output that escaped the Python repair loop is caught before the browser renders anything.

### Seam 5 -- Phase 15 studio reuses Phase 12 SpecRenderer (no stub)

WIRED

apps/web/src/app/studio/_components/spec-renderer-island.tsx is the single dynamic(ssr:false) wrapper. It imports mod.SpecRenderer from @nauta/genui/renderer (Phase 12). apps/web/src/app/studio/preview/_components/spec-renderer-island.tsx is a pure re-export (2 lines), not a second wrapper. generation-sandbox-island.tsx imports and renders SpecRendererIsland at lines 51 and 234. No stub component found. COMPONENT_REGISTRY is not serialized across the server/client boundary; SpecRenderer uses it as a default prop on the client.

### Seam 6 -- No-eval guarantee across the full genui surface

WIRED

Grep over packages/genui/src for eval, new Function, and dangerouslySetInnerHTML returns zero matches on executable code paths. All matches are in comments or test expectations. render-node.tsx uses React.createElement with a Registry allowlist lookup (only registered components can be rendered; others fall through to ErrorBoundary). Data references resolve via resolveDataRef (safe dotted-path walk, line 61). Conditions evaluate via evaluateCondition (pure value comparisons, lines 113-124). Actions resolve via ActionRegistry key lookup, never eval. The navigate action enforces relative-only href at runtime (D-15); tests confirm absolute URLs are blocked.

## E2E Flow Traces

### Flow 1: Developer enters intent, UI renders

1. Developer opens /studio (static route, server-rendered shell with REGISTRY_VERSION badge)
2. Clicks Sandbox tab, GenerationSandboxIsland mounts
3. Types intent, clicks Generate, api.genui.generate.useQuery fires
4. tRPC handler proxies to FastAPI POST /v1/genui/generate with registry_version
5. Step-0 cache check in GenerateUiSpecUseCase.execute(): SHA-256 key computed; on miss, quarantine+generate pipeline runs
6. FastAPI returns { spec, outcome, cache_hit, registry_version }
7. tRPC handler re-validates spec with SpecRootSchema.safeParse; on failure returns fallback
8. deriveGenerationState maps (isPending, outcome, cacheHit) to one of four discriminated states
9. GenerationStateChrome renders the correct state UI
10. SpecRendererIsland renders the spec via Phase 12 interpreter

Status: COMPLETE -- every step wired with code-level evidence

### Flow 2: Cache hit path

1. Same as Flow 1 steps 1-4
2. Cache key matches, SupabaseUiSpecTemplateRepository.find_by_cache_key returns CachedTemplate
3. Use case returns early with cache_hit=True, spec from store, no Bedrock call
4. increment_use_count runs best-effort (exception swallowed, non-blocking)
5. tRPC receives same response shape with cacheHit: true
6. Studio sandbox shows cache-hit state badge; spec renders via SpecRenderer

Status: COMPLETE

### Flow 3: Fallback on generation failure

1. Steps 1-5 as above; Bedrock/repair loop exhausted (<=3 attempts)
2. GenuiGeneratorAdapter returns outcome=fallback with safe fallback spec
3. GenerationAuditRepository.record() writes event best-effort
4. tRPC receives outcome: fallback, cache_hit: false
5. deriveGenerationState returns { kind: fallback }
6. Studio shows fallback badge; safe spec renders without crash

Status: COMPLETE

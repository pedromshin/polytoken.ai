<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> tRPC -> FastAPI | History reads cross the web/server boundary; FastAPI output is untrusted by the web layer |
| FastAPI -> Supabase | Service-role connection reads ui_spec_templates (bypasses deny-all RLS) |
| stored spec_json -> renderer | A stored spec may no longer parse under the current schema |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-03-I | Information Disclosure | GET /history cross-tenant | mitigate | Endpoints are read-only over ui_spec_templates with an optional importer_id eq filter; rows carry no secrets; FastAPI stays behind require_api_key (EMAIL_LISTENER_API_KEY server-side only) |
| T-16-03-T | Tampering | stored spec_json re-render | mitigate | historyById re-runs SpecRootSchema.safeParse at the web boundary and degrades to SAFE_FALLBACK_SPEC on failure (D-17) — the same safeParse the production render path uses; no spec-validation bypass |
| T-16-03-D | Denial of Service | unbounded list query | mitigate | limit clamped server-side to <=100 (FastAPI Query le=100 + adapter clamp); offset >= 0; best-effort errors -> [] never a 500 cascade |
| T-16-03-R | Repudiation | history reads | accept | Read-only, no state change to audit; existing structlog server-side logging on failure paths suffices |
| T-16-03-SC | Tampering | npm/pip installs | mitigate | No new packages installed (reuses existing fastapi/supabase/zod/trpc) — package-legitimacy gate N/A |
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runner -> Bedrock | run_eval + judge make real Bedrock calls (generation + judge); cost/timeout must be bounded |
| golden prompt -> generator | Golden prompts are trusted corpus text passed as `intent` (intent-only, raw_content="") |
| judge -> generator | The judge must never feed or influence the generation path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-02-D | Denial of Service | unbounded eval run cost | mitigate | Judge call is temperature=0, low max_tokens, asyncio.timeout (mirrors the generator); the runner caps concurrency <=3 and the judge is skippable with --no-judge; the generation path keeps its existing per-call max_tokens + timeout |
| T-16-02-E | Elevation of Privilege | judge influencing generation | mitigate | The judge is its OWN adapter using the shared client but a separate call; it never writes to the generator messages — verified by the judge being import-isolated from the generation path |
| T-16-02-T | Tampering | rubric drifts from production "valid" | mitigate | rubric.valid_spec imports the SAME load_spec_schema + MAX_SPEC_NODES/MAX_SPEC_DEPTH the generator uses, and composed reuses the exact no-placeholder phrase-set — a unit test asserts SAFE_FALLBACK_SPEC fails valid-spec |
| T-16-02-I | Information Disclosure | report contents | accept | Reports contain public corpus prompts + spec JSON + scores; no secrets (model id is a public string, no keys) |
| T-16-02-SC | Tampering | pip installs | mitigate | No new pip packages (reuses anthropic/jsonschema/structlog/pytest already in the email-listener venv) — package-legitimacy gate N/A |
</threat_model>

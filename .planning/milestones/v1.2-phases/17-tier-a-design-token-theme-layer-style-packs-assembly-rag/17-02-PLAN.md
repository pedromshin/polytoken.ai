<objective>
Build the assembly-RAG retrieval seam in the Python service: a source-agnostic `RetrievalProvider` port
that returns a ranked, scored result list, and the fullest LOCAL implementation feasible now — a
deterministic/lexical adapter that retrieves relevant catalog components, hand-authored exemplar specs,
and available `ui_spec_templates` rows, ranked top-k. The port + ranked-result contract are the drop-in
seam for the deferred FLY embedding/RRF adapter (no caller change when it lands).

retrieval engine + the hand-authored real exemplar assets (never AI-fabricated, D-12). Plan 17-04 wires
this provider into the generator's per-request injection and proves RAG-02 (the emitted spec references
the retrieved structure).

Output: `retrieval_provider.py` (port + DTOs), `genui_retrieval_provider.py` (LexicalRetrievalProvider),
`genui_exemplars.py` + committed exemplar assets, and their unit tests.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| exemplar assets → generation prompt | Exemplars are injected into the dynamic prompt portion (Plan 17-04); they are committed, hand-authored, trusted content — but must not carry instructions that smuggle non-allowlisted structure. |
| templates table → retrieval | ui_spec_templates rows are prior model output persisted to DB; treated as untrusted content when retrieved. |
| intent → lexical scorer | The intent is canonicalized/tokenized only; never executed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-10 | Tampering | exemplar assets | mitigate | Exemplars are committed, hand-authored, and CI-validated against load_spec_schema() (Task 2) — they cannot introduce non-allowlisted structure because they themselves must pass the spec schema. |
| T-17-11 | Injection | retrieved templates/exemplars → prompt | mitigate | Retrieval returns structured payloads (ids + spec dicts), NOT raw prose to interpolate as instructions; Plan 17-04 injects them in the data section and the final emitted spec is re-validated by the existing dual-LLM + Zod boundary. Documented as the contract here. |
| T-17-12 | Denial of Service | template read in retrieve() | mitigate | Template-source read is best-effort: failure is swallowed + logged; retrieval degrades to catalog+exemplars, never raises (Task 3). |
| T-17-13 | Info Disclosure | retrieved_ids logging | accept | retrieved_ids are catalog/exemplar/template ids (no PII, no raw intent) — safe to log for the D-14 RAG-02 proof. |
</threat_model>

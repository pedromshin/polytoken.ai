<objective>
Wire the Python generation pipeline to be pack-aware and RAG-grounded: a Python style-pack registry, a
cache key that includes style_pack_id, per-request injection of the retrieved subset + exemplars + active-pack
token table into the generator's DYNAMIC prompt portion (cached static block untouched), and a use case that
runs retrieval + pack selection before generation and flows style_pack_id + retrieved ids onto the result and
audit event.

Purpose: STYLE-01 (the generator is conditioned on the design system + active-pack tokens) + RAG-01/RAG-02
(retrieval runs before generation and demonstrably influences the emitted spec) + D-08 (cache correctness: a
pack swap must not serve a mis-themed cached spec).

Output: `genui_style_packs.py`, an extended `cache_key.py`, the per-request injection in
`genui_generator_adapter.py`, the orchestration in `generate_ui_spec.py`, DI wiring in `container.py`, and the
FastAPI route carrying style_pack_id.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| model output → spec envelope | The generator returns style_pack_id + per-node style props; cache + audit must record the pack actually used, and the cache key must not let a pack swap serve a stale theme. |
| retrieved exemplars/templates → generator prompt | Injected per-request; must remain DATA (quarantine discipline), never instructions that smuggle non-allowlisted structure. |
| style_pack_id at the FastAPI route | Untrusted request field; validated against KNOWN_STYLE_PACK_IDS. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-20 | Tampering | exact-cache key | mitigate | style_pack_id is a key dimension (Task 1); two packs → two keys; a pack swap can never serve a mis-themed cached spec (Task 1 + Task 3 tests). |
| T-17-21 | Tampering | generator system block | mitigate | RAG + token-table injection is per-request (DYNAMIC user turn) only; _build_system_blocks stays byte-identical (cache_control stability test) — preserves COST-01 and prevents cache-prefix poisoning. |
| T-17-11 | Injection | retrieved exemplars/templates → prompt | mitigate | Retrieved data is framed as DATA inside the user turn (not system instructions); the dual-LLM quarantine + the emitted spec's Zod/jsonschema re-validation (existing) catch any non-allowlisted structure; the token table is built from the pack registry, not model input. |
| T-17-04 | Spoofing | style_pack_id at the route | mitigate | routes/genui.py validates style_pack_id against KNOWN_STYLE_PACK_IDS; unknown rejected; use case also defaults None → DEFAULT_STYLE_PACK_ID. |
| T-17-13 | Info Disclosure | retrieved_ids on audit | accept | retrieved_ids are catalog/exemplar/template ids (no PII, no raw intent); intent stays a hash on the audit row (existing privacy posture). |
</threat_model>

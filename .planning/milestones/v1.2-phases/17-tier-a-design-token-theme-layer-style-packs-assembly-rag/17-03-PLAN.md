<objective>
Make style packs VISIBLE: mount a themed CSS-variable wrapper in the shared renderer so the active pack's
color/radius/shadow/font tokens drive `@nauta/ui` automatically, resolve per-node token props to pack CSS
vars, add the explicit style-pack dropdown + Auto/Surprise control to the /studio sandbox, and thread
`style_pack_id` through the tRPC generate proxy (Zod-validated at the web boundary).

Purpose: This is the consumption half of STYLE-01/STYLE-02 — the same intent under two packs must visibly
differ in look-and-feel (success criterion 2), driven by a studio control (D-04), not the model. The
themed wrapper is the single point where alias → CSS-var resolution happens, so the renderer never
interpolates model text into CSS.

Output: `ThemedRoot` wrapper, an extended SpecRenderer, the studio pack dropdown + Auto/Surprise, and the
tRPC proxy carrying stylePackId.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| model spec → renderer CSS | Per-node token props + style_pack_id come from model output; the renderer must map alias→curated CSS var, never interpolate model text into a style string. |
| studio input → tRPC → FastAPI | stylePackId is user/studio-selected; validated against STYLE_PACK_IDS at the web boundary before forwarding. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-02 | Elevation of Privilege | ThemedRoot / per-node style resolution | mitigate | Style values are built ONLY from the pack's curated resolvedVars and alias→var map; a model token prop selects a var by enum alias, never supplies a CSS value — no CSS injection surface (Task 1 test asserts alias→var, never raw string). |
| T-17-04 | Spoofing | stylePackId at the web boundary | mitigate | GenerateInput validates stylePackId against z.enum(STYLE_PACK_IDS); unknown rejected; ThemedRoot also falls back to default (Task 1/2). |
| T-17-14 | Info Disclosure | EMAIL_LISTENER_API_KEY | mitigate | Read via getListenerConfig() server-side at call time; never NEXT_PUBLIC_ (existing posture preserved). |
</threat_model>

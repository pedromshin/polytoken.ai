<objective>
Establish the TypeScript token/theme contract layer: a library of >=5 distinct hand-authored W3C-DTCG
style packs (default = the current Nauta teal theme), a fourth TOKEN allowlist that constrains per-node
style props to pack-defined token aliases, an optional `style_pack_id` on the spec envelope, and the
re-emitted Bedrock artifacts (behind the CI drift gate).

Purpose: This is the machine-readable design system the generator is conditioned on (STYLE-01) and the
"constrain to token aliases, never free-form" safety+quality lever (STYLE-03). Everything downstream
(renderer theming, Python pack registry, eval distinctiveness) consumes these contracts. Building the
contracts first prevents downstream plans from a scavenger hunt.

Output: `packages/genui/src/theme/*` (DTCG packs + token types + allowlist), an extended spec schema
(`style_pack_id` envelope field + per-node token-props), the fourth allowlist re-export, and regenerated
`artifacts/spec.schema.json` + `artifacts/genui-prompt.json`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| model output (Bedrock) → spec schema | The generator now emits per-node style props; a non-allowlisted alias or raw value is untrusted input crossing into the renderer. |
| model output → spec envelope | style_pack_id in the emitted spec is untrusted; an unknown id must not select a phantom pack. |
| committed artifacts → Bedrock tool grammar | spec.schema.json becomes the constrained-decoding grammar; a malformed/stale schema breaks generation (v1.1 live-400 class). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Tampering | per-node style props (token-props-schema.ts) | mitigate | TokenPropsSchema values are `z.enum(TOKEN_ALIASES)` — raw hex, calc(), url(javascript:) and any non-allowlisted alias are rejected by safeParse BEFORE render (Task 2 negative tests are the proof). |
| T-17-02 | Elevation of Privilege | CSS-injection via token value | mitigate | No free-form string ever becomes a style value; the renderer (Plan 17-03) maps an alias → a known CSS variable, never interpolates model text into CSS. Enforced here by making the schema value-set closed (enum). |
| T-17-04 | Spoofing | style_pack_id envelope | mitigate | SpecRootSchema validates style_pack_id against z.enum(STYLE_PACK_IDS); unknown id is rejected (Task 2 test). getStylePack() also falls back to the default baseline, never throws. |
| T-17-06 | Tampering | spec.schema.json (Bedrock grammar) | mitigate | CI drift gate (Task 3) + the existing additionalProperties/root-type load guard keep the emitted grammar valid and in sync with the Zod source. |
</threat_model>

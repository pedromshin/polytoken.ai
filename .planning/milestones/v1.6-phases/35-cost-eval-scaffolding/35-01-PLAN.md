<objective>
Add a per-round cost ceiling to the FOUND-3 `CostCircuitBreaker` — DISTINCT from the existing
per-turn/per-session/per-day caps — and wire it into Phase 34's bounded tool-round loop at BOTH the
point a round is still streaming (mid-round, inside `_stream_round_deltas`/`_terminal_status_for`)
and the point a round just finished executing a server tool (the round boundary, inside
`_run_server_tool_round`). A trip at either point aborts the turn `cost_capped`, reusing Phase 34's
existing "never silent" persistence path so the partial text/parts already accumulated are never
dropped.

Purpose: COST-05 — a per-round ceiling is a materially different risk control than a per-turn cap
(a single expensive round inside a cheap-looking multi-round turn should be caught DURING that
round, not only after the whole turn's cumulative spend crosses the per-turn line). Built and
provable against Phase 34's echo stub, before Phase 36/37 wire real tools onto this same loop.
Output: `CostCircuitBreaker.should_abort_round`, `COST_CAP_PER_ROUND_USD` setting (default $0.15),
round-scoped checks wired into the round loop, new tests proving distinctness from the per-turn cap.
</objective>

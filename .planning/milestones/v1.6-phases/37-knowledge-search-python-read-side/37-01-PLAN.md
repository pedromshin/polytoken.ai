`extracted_only` view that makes non-EXTRACTED free text structurally unreachable (belt 1), a pair

these two repository methods directly; it cannot be built or tested meaningfully until this plan's
SQL and Python contracts exist.

live-verify script proving the view's tier-based text nulling against real Postgres with seeded
three-tier data, and `search_nodes`/`expand_neighbours` on `KnowledgeGraphRepository` +
`SupabaseKnowledgeGraphRepository`, each with call-shape/logic unit tests (no live DB).

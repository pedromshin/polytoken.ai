<objective>
Render Phase 39's TUI-01/TUI-02 visible surface: a live "searching…" activity row while a tool round
streams, a quiet collapsed result row once it settles, and every tool result's citations as chips
through ONE shared `<ProvenanceLink kind id />` primitive — the same primitive Phase 41's
knowledge-preview node will later consume. Also fixes a real, already-live SSE-event naming collision
(39-UI-SPEC.md's "SSE / Part Contract" section): the client's existing `applyRunEvent` reducer
currently mis-folds a real server-tool round's persisted `tool_call` event into a permanently-stuck,
empty `interactive_widget_streaming` skeleton part, since that reducer was built for the emit_ui_spec/
interactive-widget streaming case and keys off a `partial_json` field server-tool events never carry.

Purpose: TUI-01 requires a visible in-progress affordance; TUI-02 requires one shared citation-chip
primitive. Both depend on `use-chat-stream.ts`'s type contracts being correct FIRST (interface-first
ordering) — the 2 new row components and the message-turn.tsx wiring are built against those contracts.

Output: `use-chat-stream.ts` extended (types + reducer + collision fix), a new standalone
`ProvenanceLink` primitive, 2 new chat-transcript row components, and `message-turn.tsx`'s part switch
wired to render all 3 new part types.
</objective>

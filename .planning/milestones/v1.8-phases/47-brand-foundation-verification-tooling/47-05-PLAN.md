Reuse from 47-04: apps/web already has @playwright/test + a webServer that boots `npm run dev` at
http://localhost:3000. The capture harness should reuse that webServer wiring so a single command boots the app
and captures. Keep the capture spec SEPARATE from the two assertion specs (code-island, auth-redirect) so
`test:e2e` (assertions) and `screenshot:review` (capture) do not run each other — a dedicated
`playwright.screenshot.config.ts` whose `testDir`/`testMatch` targets only `e2e/screenshot-review.spec.ts` is the
clean split (it can import and extend the base config's webServer/baseURL).

Which surfaces are auth-gated: with no Supabase session, the middleware redirects protected routes to
`/login?redirectTo=...` (see auth-redirect.spec.ts). login is public; the inbox `/`, `/chat`, `/knowledge`,
`/studio`, `/settings/forwarding` will redirect when signed out — so the harness records "redirected to /login
(no session)" for those and captures whatever renders (the login target), rather than faking a session.

.planning/ui-reviews/ exists with a .gitignore that ignores *.png/*.webp/*.jpg/etc. — write PNGs there; they stay
uncommitted. The index.md is NOT ignored, but it must be a pure manifest (surface/viewport/auth-status/filename)
carrying NO env values or secrets.

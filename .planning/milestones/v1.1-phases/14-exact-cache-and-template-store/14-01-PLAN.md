<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Supabase client (anon/authenticated role) → ui_spec_templates | A direct PostgREST/Supabase-JS client could read or write cached specs, bypassing the FastAPI service's importer_id/catalog_id scoping. The FastAPI service connects as postgres/service-role (bypasses RLS by design); anon/authenticated must be denied. |
| Generation pipeline → spec_json column | The persisted spec is data. A non-validated or fallback spec written here would later be served as a cache hit (poisoning). The validation_status CHECK + the Plan 14-03 validated-only write gate close this. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Information Disclosure | ui_spec_templates direct read by anon/authenticated (cross-tenant cache leak) | mitigate | RESTRICTIVE deny-all RLS for anon + authenticated in 0022 (D-20). Acceptance test: pg_policies lists both deny-all policies; RLS enabled. |
| T-14-02 | Tampering | anon/authenticated forging/overwriting a cached spec row | mitigate | Same RESTRICTIVE deny-all RLS WITH CHECK(false) blocks writes (D-20). |
| T-14-03 | Tampering (cache poisoning) | a non-'validated' spec persisted and later served | mitigate | DB-level `validation_status IN ('validated')` CHECK constraint (D-10/D-11) rejects any other status at the boundary, independent of the Python write path. Acceptance test: INSERT with 'promoted' raises CHECK violation. |
| T-14-04 | Tampering | migration applied non-idempotently / drifting from sibling tables | accept→mitigate | `IF NOT EXISTS` guards + DROP POLICY IF EXISTS (mirrors 0020/0021); rationale: re-run safe, matches the established migration convention. |
| T-14-SC | Tampering | npm/drizzle-kit dependency (no NEW package installed; drizzle-kit ^0.31.1 already present) | accept | No new package installed this plan — only the existing drizzle-kit CLI is invoked. No legitimacy gate required. |
</threat_model>

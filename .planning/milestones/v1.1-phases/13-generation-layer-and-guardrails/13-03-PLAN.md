<objective>
Build the FastAPI generation service that turns a trusted intent + untrusted content into a validated `SpecRoot`: a dual-LLM quarantine (Call A extracts enum-constrained structured data; Call B generates the spec and never sees raw prose), a bounded repair loop (≤3 attempts, Haiku→Sonnet 4.6 escalation on the last), a safe fallback on persistent failure, cost controls on every Bedrock call (max_tokens, asyncio timeout, temperature 0, cache_control), the audit-row write, and the `POST /v1/genui/generate` endpoint behind X-API-Key.

This honors D-01: the Bedrock call lives in the Python service on ECS (task IAM role), translating the research's AI-SDK `Output.object` mechanism onto the proven `AsyncAnthropicBedrock` forced-tool-use pattern already in `autofill_adapter.py`.

Purpose: success criteria 1, 2, 4, and the caching clause of 5 — the core generation + guardrail behavior.
Output: quarantine + generator adapters, the generate use case, the endpoint, DI wiring, new settings, and TDD tests.
</objective>

<objective>
Establish the TypeScript contract layer that BOTH the Python generator and the web renderer target: the three allowlists (component-type, tRPC-procedure, action-href) enforced at the Zod schema level, the static safe-fallback spec, and the build artifacts (`spec.schema.json` + compact-catalog/procedures `genui-prompt.json`) that bridge the Zod single-source-of-truth across the TS/Python split (D-03/D-22).

This is the contract Wave 2 (Python generation) and Wave 3 (web proxy + binding layer) consume. It is self-contained TypeScript inside `packages/genui` with no Python and no web-app file touches, so it runs in parallel with the audit-table plan (13-02).

Purpose: A single `safeParse` of the Zod schema must reject all three violation classes (unregistered type, non-allowlisted procedure, non-relative action), and the Bedrock-facing artifacts must never drift from the Zod source.
Output: extended spec schema, allowlist schemas, `SAFE_FALLBACK_SPEC`, an emit script + committed artifacts, and a CI freshness test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Zod schema → renderer | The schema is the single validation gate the renderer trusts; any node/binding/action not permitted here must be rejected before render (D-15). |
| Zod source → Bedrock JSON Schema artifact | The emitted spec.schema.json crosses to the Python service; if it drifts from the Zod source the two validators disagree and a spec could pass one but not the other. |

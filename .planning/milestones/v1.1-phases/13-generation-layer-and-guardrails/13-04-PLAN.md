<objective>
Wire the web side: a `genui` tRPC router that proxies to the FastAPI `POST /v1/genui/generate` (Phase 4-10 pattern), re-validates the returned spec against the canonical Zod `SpecRootSchema` at the web boundary (the schema the renderer trusts), and falls back to `SAFE_FALLBACK_SPEC` on any validation failure — never raw model output. Plus the binding/action layer: an `ActionRegistry` that binds query + setState + navigate handlers (with a runtime relative-href re-check) and deliberately leaves the `mutate` handler unregistered — the SEAM-02 "mutation path exists but is empty" guarantee.

GEN-04 progressive streaming lands non-streaming this phase per the CONTEXT [JUDGMENT CALL]: the safety model (validate-before-render) argues against rendering partial specs, so the procedure buffers + validates the full spec before returning.

Purpose: success criterion 3 (web-boundary allowlist rejection) + success criterion 5 (query-only binding, empty mutate seam) + GEN-03 fallback at the trust boundary.
Output: the genui tRPC router + Zod re-validation, the ActionRegistry handlers, the SpecRenderer provider wiring, and vitest tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FastAPI spec → web tRPC procedure | The spec from the Python service is re-validated against the canonical Zod schema before the renderer trusts it. |
| Spec action → runtime handler | The ActionRegistry resolves declared actions to handlers; navigate must re-verify the relative-href invariant before acting. |
| mutate action → (no handler) | The mutate branch validates in schema but binds to no handler in v1.1 (SEAM-02). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-18 | Tampering | spec returned from FastAPI | mitigate | SpecRootSchema.safeParse at the web boundary (D-08); failure → SAFE_FALLBACK_SPEC; the renderer only receives a Zod-valid spec; test asserts unregistered-type/non-relative-navigate specs are rejected. |
| T-13-19 | Information Disclosure | raw model output / error detail to client | mitigate | Never return raw invalid spec; transport errors return a friendly fallback message, detail logged server-side (GEN-03/§3d); test asserts no leaked detail. |
| T-13-20 | Tampering (open redirect/XSS) | navigate href at runtime | mitigate | navigate handler re-applies the relative-only guard before navigating; javascript:/external blocked at runtime (SAFE-04/D-15); test asserts no navigation on a malicious href. |
| T-13-21 | Elevation of Privilege | mutate action wired by accident | mitigate (seam) | No mutate handler is registered; useActionRegistry returns noop (SEAM-02/D-23); test asserts the mutate id resolves to noop. |
| T-13-22 | Elevation of Privilege | query-refresh to arbitrary procedure | mitigate | query-refresh validates the procedure against ALLOWED_PROCEDURES before calling refetch (SAFE-03). |

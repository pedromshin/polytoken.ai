<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| spec `type` key → registry lookup | A spec node's `type` is looked up in COMPONENT_REGISTRY. An unregistered/hallucinated type must not reach a component (Phase 13 LLM threat; seamed here). |
| catalog change → cache key | REGISTRY_VERSION is the value Phase 14 keys its cache on; a silent catalog drift must change the version. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-05 | Tampering | unknown/hallucinated `type` key (Pitfall 2) | mitigate | Allowlist `RegisteredTypeSchema = z.enum(Object.keys(COMPONENT_REGISTRY))` rejects unregistered types at validation; `UnknownComponentPlaceholder` is the non-throwing render fallback (D-06). Only registry-keyed components can render. |
| T-12-06 | Tampering | manifest propsSchema drifts from real component (Pitfall 8) | mitigate | CTLG-04 example test asserts every example parses its own schema; a drifted manifest fails CI (`npm run test -w @nauta/genui` exits non-zero). |
| T-12-07 | Elevation of Privilege | LLM-set dangerous prop (onClick, type=submit, dangerouslySetInnerHTML) | mitigate | UI-SPEC §11 locked-props are excluded from every propsSchema; `dangerouslySetInnerHTML`/`ref`/`key` are categorically absent from all manifest entries (grep-verified). `button.onClick` is ActionRegistry-only; `button.type` locked to "button". |
| T-12-08 | Information Disclosure / stale cache | registry version not bumping on change | mitigate | SHA-256 content-hash over sorted type keys + serialized schema surface (D-07) flips on ANY catalog change → Phase 14 cache auto-invalidates with no manual flush (CACHE-04 seam). Determinism + sensitivity asserted in test block 4. |
| T-12-SC | Tampering | npm installs | accept | No new third-party runtime packages — @nauta/ui is an existing workspace dep, crypto/vitest are builtins/existing. No package-legitimacy checkpoint required. |
</threat_model>

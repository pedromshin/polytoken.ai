## Live UAT findings (2026-06-13, during this audit)

User live-tested in the browser and reported: reprocess console error, "every box
is pending", and "cant add region". **Root cause: single local-env gap** — root
`.env.local` (loaded by the web dev server) lacked `EMAIL_LISTENER_URL` +
`EMAIL_LISTENER_API_KEY` (added to the example only in Phase 6, after the file was
created). Reads use Postgres/Drizzle directly and worked; every WRITE proxies to
FastAPI via `getListenerConfig()`, which threw "not configured" → reprocess /
accept / add-region / autofill / confirm all failed. "Every box is pending" is the
**correct** initial state: segmentation-proposed regions are born `pending`
(propose_regions.py:234) and the human Accept transitions them `pending→candidate`
(edit_region.py) — the user simply couldn't accept any while the write proxy was
down. Fix applied: added both vars to `.env.local` (untracked); requires a web
dev-server restart. No code defect.
